# 实验报告：Whitted-Style 光线追踪（Taichi）

## 一、实验目标

1. 理解 Ray Casting 与 Ray Tracing 的本质区别。  
2. 掌握通过次级射线实现硬阴影与理想镜面反射。  
3. 学习将递归光线追踪改写为适合 GPU 的迭代模式。  

## 二、实验原理

主光线从相机发出，命中场景后执行：

1. **阴影测试**：从交点向点光源发射暗影射线。若中途被其他几何体遮挡，则该点仅保留环境光。  
2. **材质分支**：  
   - 漫反射材质：按 Phong 模型计算颜色并终止该路径；  
   - 理想镜面材质：按反射公式计算方向并继续弹射。  

反射向量公式：

\[
\mathbf{R} = \mathbf{L}_{in} - 2(\mathbf{L}_{in}\cdot\mathbf{N})\mathbf{N}
\]

## 三、实验环境

- 语言：Python 3.8+
- 框架：Taichi（GPU 优先，失败回退 CPU）
- 文件：`ray_tracing_taichi.py`

## 四、实验内容与实现

### 1. 隐式场景与材质 ID

- Ground Plane：`y=-1.0`，法线 `(0,1,0)`，棋盘格纹理  
- 红色漫反射球：`(-1.5,0,0), r=1.0`  
- 银色镜面球：`(1.5,0,0), r=1.0`  
- 材质标签：`MAT_DIFFUSE=0`, `MAT_MIRROR=1`

### 2. 迭代式多次弹射

每个像素路径维护：

- `throughput`（初值 1.0）
- `final_color`（初值 0.0）

在固定上限循环中追踪路径，镜面命中继续弹射，漫反射命中累积后终止。该方式规避了 GPU 不擅长递归的问题。

### 3. 硬阴影与精度修复

暗影射线和反射射线都使用偏移起点：

\[
\mathbf{P}_{new}=\mathbf{P}+\epsilon\mathbf{N},\ \epsilon=10^{-4}
\]

成功避免了自相交导致的 Shadow Acne。

### 4. UI 交互

窗口内提供实时滑动条：

- `Light X / Light Y / Light Z`
- `Max Bounces`（1~5，默认 3）

可直观看到阴影移动与反射深度变化。

## 五、实验结果与现象

1. 当 `Max Bounces = 1` 时，镜面球基本无二次反射内容。  
2. 当 `Max Bounces >= 2` 时，可观察到镜中反射场景。  
3. 调整光源位置时，地面与球体阴影会实时变化，形成硬边界。  
4. 去掉 `epsilon` 偏移会出现明显黑点噪声；恢复偏移后画面稳定。  

## 六、结论

本实验完成了 Whitted-Style 光线追踪的核心最小系统：主光线、暗影射线、反射射线与迭代弹射。通过材质分支与吞吐量累计，能够在实时交互中清晰展示全局光照中“路径传播”的概念，并验证了 GPU 编程中“以循环替代递归”的工程实践价值。
